回溯算法DFS

  • 回溯算法框架
  • 解决一个回溯问题，实际上就是一个决策树的遍历过程
1、路径：也就是已经做出的选择。
2、选择列表：也就是你当前可以做的选择。
3、结束条件：也就是到达决策树底层，无法再做选择的条件。
  • 46. 全排列问题
  • 给定一个不含重复数字的数组 nums ，返回其所有可能的全排列。你可以按任意顺序返回答案。
比方说给三个数 [1,2,3]
[2] 就是「路径」，记录你已经做过的选择；[1,3] 就是「选择列表」，表示你当前可以做出的选择；「结束条件」就是遍历到树的底层，在这里就是选择列表为空的时候。
对链表使用 contains 方法需要 O(N) 的时间复杂度。不管怎么优化，都符合回溯框架，而且时间复杂度都不可能低于 O(N!)，因为穷举整棵决策树是无法避免的。这也是回溯算法的一个特点，不像动态规划存在重叠子问题可以优化，回溯算法就是纯暴力穷举，复杂度一般都很高。

class Solution {

    List<List<Integer>> res=new LinkedList<>();

    public List<List<Integer>> permute(int[] nums) {
        LinkedList<Integer> track=new LinkedList<>();
        backtrack(nums,track);
        return res;
    }

    void backtrack(int[] nums,LinkedList<Integer> track){
         //判断是否到达结束条件
         if(track.size()==nums.length){
            res.add(new LinkedList(track));
            return;
         }

         for(int i=0;i<nums.length;i++){
             //如果路径已包含该数，则跳过
             if(track.contains(nums[i])){
                 continue;
             }
             //做选择，添加到路径
             track.add(nums[i]);
             //回溯，下一层决策数
             backtrack(nums,track);
             //取消选择，回到上一层
             track.removeLast();
         } 
    }
}
  • 51. N皇后问题
  • n 皇后问题研究的是如何将 n 个皇后放置在 n×n 的棋盘上，并且使皇后彼此之间不能相互攻击。皇后可以攻击同一行、同一列、左上左下右上右下四个方向的任意单位。
  • 给你一个整数 n ，返回所有不同的 n 皇后问题的解决方案。
  • 每一种解法包含一个不同的 n 皇后问题的棋子放置方案，该方案中 'Q' 和 '.' 分别代表了皇后和空位。
